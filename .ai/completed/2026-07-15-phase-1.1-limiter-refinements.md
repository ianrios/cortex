## Problem

Ian ratified three behaviors Phase 1 lacks: (1) per-file exemptions
allowed everywhere but WARNED as visible debt, (2) a superadmin scan
ignoring the ignore/exempt lists, (3) broader default ignores.

## Changes

0. **Rename `exemptDirs` → `archiveDirs`** across src, tests, README,
   USAGE.md, root brickwall.config.json, EXTRACTION_PLAN mentions.
   Hard rename, no alias (pre-publish, zero consumers).
1. **Warnings channel** (checks.ts, run.ts, cli.ts, index.ts): RunResult
   gains `warnings`; `Warning = { type: 'exemption-debt' |
   'stale-exemption'; message: string }` (mirrors Violation). Warnings
   NEVER affect the exit code. Human output prints a `⚠` block after
   the verdict; `--json` becomes `{ violations: [...], warnings: [...] }`
   — BREAKING (was bare array; acceptable pre-publish, do it now).
   Deterministic ordering: warnings follow config-entry order.
2. **exemptFiles applies to code-size** (doc checks already honor it) —
   but NOT to eslint-disable: the disable ban stays universal outside
   archiveDirs (inline escape hatches are the enemy; a data manifest may
   be long but may never carry eslint-disable). Warning semantics:
   - Scanned set = mdFiles ∪ codeFiles as the checks see them (test
     files excluded), minus files under archiveDirs.
   - Custom entry (not in default exemptFiles) matching ≥1 scanned file
     → one `exemption-debt` warning enumerating ALL matched files
     (basename matching means one entry can shield many; the warning
     must show the real blast radius).
   - Custom entry matching 0 scanned files → `stale-exemption` warning.
   - Default entries (`CHANGELOG.md`) always silent, matching or not.
3. **`--all` flag** (walk.ts, run.ts, cli.ts): fs walk (git ls-files
   cannot see gitignored files) with ONLY `node_modules` and `.git`
   skipped (ratified DECISION — supersedes the earlier ".git only"
   phrasing); archiveDirs and exemptFiles disabled too. Exit codes keep
   normal semantics. Human header notes "full-scope audit".
   Implementation note: walkFs recursion prunes bare names only; slash
   entries are filtered post-hoc. README frames --all as an audit view
   for humans/agents, NOT a CI gate: build output (dist/coverage) and
   test fixtures WILL fire — that is truthful full-scope output.
4. **Defaults**: ignoreDirs += `.vscode`, `.codex`, `.cursor` (in
   config.ts DEFAULT_CONFIG; sync README's embedded default block).
5. **Docs**: package README (warnings, --all, new JSON shape, renamed
   key, default-list sync) and docs/USAGE.md (verify accuracy; rename).
